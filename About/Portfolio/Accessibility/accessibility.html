Website Accessibility Features

1. Language Declaration

The website specifies the language of the content using the <html lang="en"> tag. This helps screen readers properly interpret the content and apply the correct language settings for users, ensuring an accurate reading experience.

2. Skip to Main Content

Offering a Skip to Main Content link at the top of each page. This allows keyboard and screen reader users to bypass repetitive navigation and go straight to the primary content. This is especially helpful for users who cannot use a mouse or trackpad.

3. ARIA Attributes for Enhanced Accessibility

Using ARIA (Accessible Rich Internet Applications) attributes to ensure dynamic content and user interface elements are accessible. For instance:

  • aria-expanded="false" on the burger menu button helps screen readers announce the state of the menu (whether it’s open or closed).
  • aria-controls="about-menu" links the button to the menu, so screen readers understand which part of the page is controlled by the button.
  • aria-label="Toggle navigation" provides a label for the button, giving screen reader users a clear description of its function.

4. Main Content Structure

Also using role="main" to identify the primary content area. This allows screen readers to easily navigate to the main content, ensuring a smoother browsing experience for users relying on assistive technologies.

5. Clear Headings and Content Structure

Headings, such as <h2> and <h3>, are used to logically structure the content. This makes it easy for screen reader users to navigate through sections quickly.

6. Descriptive Links

Ensuring all links have descriptive text so users can easily understand where they will be taken. For example, Projects clearly describes that it links to the Projects section of the site.

7. Icons with Text

Icons are paired with descriptive text, making them accessible to both sighted users and those relying on screen readers. For example, the "About" link has both an icon and the word "About" to describe its purpose.

8. Accessible Footer

The footer of the site uses role="contentinfo" to identify it as containing important information, such as copyright and accessibility statements. This helps screen readers easily locate this section of the page.

9. Focus Management

Ensuring that interactive elements like the burger menu button are focusable and have proper focus management. This allows keyboard users to navigate through the page easily and interact with all elements.

10. WCAG Compliance

This website follows WCAG (Web Content Accessibility Guidelines) to ensure that it is accessible to everyone, including people with disabilities. By adhering to these guidelines,I strive to create an inclusive web experience.
